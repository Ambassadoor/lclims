Cautions and restrictions when using the bpoContinue flag

Note: This flag is retained in order to maintain compatibility.
           The use of StartPrint(), PrintOut(), and EndPrint() is recommended instead.
  • Each time the normal DoPrint method is called, a print job is generated.
    Specifying the bpoContinue flag keeps the completion of the print job open until the next call.
    If the same flag is also specified in the next DoPrint, the completion of the print job continues to be kept open.
  • If the bpoContinue flag is set, parameters excluding the other flags of the print option (dwOption) and the copy counter of print parameter (szOption) have their settings at the time the DoPrint method is first called being the valid ones, and are ignored during the DoPrint method's second and subsequent calls until the completion of the print job.
  • Once the bpoContinue flag is set in the DoPrint method, the print job can be completed (=actual start of printing) in any of the following ways.
    1. The bpoContinue flag is not set during the DoPrint method's subsequent calls. 
    2. Calling the Close method.
    3. A template with a different printer driver and paper size (label width in the case of labels) is specified and called using the Open method.
    4. Releasing the b-PAC object.

  • After setting the bpoContinue flag and calling the DoPrint method, it will not enter the process of waiting for the various types of end-user events such as data input or selection, until the print job is completed (=actual start of printing) in any of the ways described in a, b, c, and d.
  • The maximum number of times (number of pages that can be outputted in a single print job) that the DoPrint method with the bpoContinue flag specified can be called up depends on each printer's maximum print pages or the PC's resources.
