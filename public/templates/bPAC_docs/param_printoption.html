enum bpac::PrintOptionConstants

Print options

If the driver's print option settings are not to be adopted and the printing options are to be set, specify the settings by combining the following flags.
If the settings are to be adopted, specify "0".
(With Visual Basic®, when the flag is specified with a direct numeral, not an enumerating value that begins with bpo, the half cut flag 0x200, for example, is specified as &H200.)

For details on combining cut options, refer to "Printout samples and cut option combinations for each major device type".

Value of enumerator:
Driver Flag Description
 

Label printer
 (P-touch)

 

 

 bpoDefault
(0x0)		 The current drive settings are adopted.
bpoAutoCut
(0x1)		 Autocut is applied.
bpoCutPause
(0x1)		 Pause to cut is applied.
 Valid only with models not supporting the auto cut function.
bpoCutMark
(0x2)		 Cut mark is inserted.
 Valid only with models not supporting the auto cut function.
bpoHalfCut
(0x200)		 Executes half cut.
bpoChainPrint
(0x400)		 Continuous printing is performed.
 The final label is not cut, but when the next labels are output, the preceding blank is cut in line with the cut option setting.
bpoTailCut
(0x800)		 Whenever a label is output, the trailing end of the form is forcibly cut to leave a leading blank for the next label output.
bpoSpecialTape
(0x00080000)		 No cutting is performed when printing on special tape.
 Valid only with PT-2430PC.
bpoCutAtEnd
(0x04000000)		 "Cut at end" is performed.
bpoNoCut
(0x10000000)		 No cutting is performed.
 Valid only with models supporting cut functions.
bpoMirroring
(0x4)		 Executes mirror printing.
bpoQuality
(0x00010000)		 Fine-quality printing is performed.
bpoHighSpeed
(0x01000000)		 High-speed printing is performed.
bpoHighResolution
(0x02000000)		 High-resolution printing is performed.
bpoColor
(0x8)		 Color printing is performed.
bpoMono
(0x10000000)		 Monochrome printing is performed.
 Valid only with models supporting the color printing function.
bpoContinue
(0x40000000)		 Combines with printing for the following DoPrint( ) so that it is a single print job.
 As a result, when the next DoPrints are called up, the front margins are not output.
